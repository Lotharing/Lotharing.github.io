Git命令提交项目到Github

记录使用Git命令把项目从本地提交到Github中

2018-12-01
#《Git》
Git操作指南

Git本是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理 Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

准备工作

  • 首先登陆进Github账户中，如果没有去注册Github官网

  • 安装Git工具Git

Github操作

  • 进入Github首页，点击New repository

  • 填写相应信息后点击Creat即可，信息包含如下

  1. Repository name: 仓库名称

  2. Description(可选): 仓库描述介绍

  3. Public, Private : 仓库权限（公开共享，私有或指定合作者）

  4. Initialize this repository with a README: 添加一个README.md

  5. gitignore: 不需要进行版本管理的仓库类型，对应生成文件.gitignore

  6. license: 证书类型，对应生成文件LICENSE

  • 点击Clone or download , Copy该仓库的地址

  • Git命令操作

  • 点击想要上传的项目右键，选择Git Bash shell

  • 输入代码

git clone git@github.com:Lotharing/GIt.git(仓库地址)

  • 成功之后，会在项目中出现一个与仓库名相同的clone到本地的文件夹

  • 把需要传进仓库的文件都放进该文件夹

  • cd FileName FileName代表clone下来的文件夹名称

  • git add . （注：别忘记后面的. 此操作是把FileName文件夹下面的文件都添加进来）

  • git commit -m “提交信息” （注：“提交信息”里面换成你需要，如“第一次修改”）

  • git push -u origin master （注：此操作目的是把本地仓库push到github上面，此步骤需要你输入帐号和密码）

  • 完工，查看Github账户中需要上传的文件已经到达指定仓库，提交信息在文件后边
  • 该文章只记录了使用Git命令把项目从本地push到Github仓库中去
