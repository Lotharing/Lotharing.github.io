Git+Hexo+Github-Page搭建静态博客

教程记录了使用Hexo+Github-Page+Git工具搭建个人博客的过程

Posted by LuTong.Zhao on 2018-09-03
#《教程》
搭建个人服务教程

这篇文章记录搭建过程中的难点和问题的解决，也让我熟悉了GIt的使用，以前基本都是用GIthub客户端对仓库上传修改
现在会使用了Git根据SSH秘钥通过命令来上传修改，也让我在这方面有一定的提升

一丶准备工作

1.下载note.js并安装

2.下载Git工具并安装

3.利用Git安装Hexo

本地文件夹hexo

$ cd d:/hexo

待安装hexo

$ npm install hexo-cli -g

建立blog文件夹存放静态博客资源

$ hexo init blog

进入blog文件夹

$ cd blog

安装命令

$ npm install

生成本地Public文件，在Blog文件夹目录下

$ hexo g
$ hexo generate

运行本地服务器

$ hexo s
$ hexo server

4.在Git中使用hexo g 命令自动根据主题生成Public文件，该文件就是部署前的文件

5.在Git中使用hexo s 命令可以用本机的4000端口预览，出现banner图中一个hexo主题，则准备工作已经完成

二丶主题配置

1.Hexo官网提供很多主题，挑选完成后进入该主题作者的Github仓库

2.Dowload该主题，修改文件夹名称，copy在hexo/Blog/themes中

3.找到hexo目录下的_config.yml配置文件，找到theme并修改属性值为你自定义的主题文件夹名称

4.命令操作

清除原生成的Public文件

hexo clean

生成修改主题后的Public文件

hexo g

部署在本地端口4000上

hexo s

5.关于主题修改

5.1 根目录下的_config.yml是配置标题，副标题，作者，以及部署在Github上的Url连接，和SSH仓库传输协议秘钥等（密钥配置随后中说明）

5.2 themes下的_config.yml是主题配置包括导导航栏，Github连接，FaceBook等个人社交连接，主题Banner图，以及Tags标签和分类等

5.3 source下的md文件就是笔者当下写的文章，根据MarkDown规范去写就行，hexo可以根据md文件自动生成可视化网页，自行查看MarkDown书写规范

5.4.1 新建Md文件输入 hexo new ticname 就会在 source下的_post中出现md文件，按照MarkDown规范编辑即可

5.5 关于Tags（标签）和Gategores（分类）的配置

创建一个Tag

hexo new page “tags”

编辑新创建的页面并将页面MD文件类型设置tags为前面的内容。

title: All tags
type: “tags”

添加tags到主题菜单中_config.yml

menu:
Home: /
Archives: /archives
Tags: /tags

创建一个名为的页面 categories

$ hexo new page “categories”

编辑新创建的页面并将页面MD文件类型设置categories为前面的内容。

title: All tags
type: “categories”

添加Categories到主题菜单中_config.yml：

menu:
Home: /
Archives: /archives
Categories: /categories

三丶SSH秘钥

1.Git中输入命令(一直Enter键)

$ ssh-keygen -t rsa -C “github邮箱

2.本地用户User生成秘钥文件 .pub扩展名的公共秘钥

3.在Github账户中Settings -> SSH and GPG keys -> New SSH Key ->.pub的内容粘贴进去保存

4.在_config.yml的最下方找到depoly并修改

deploy:
type: git
repository: git@github.com:YourGithubAccountName/CangkuName
例如：repository: git@github.com:Lotharing/Lotharing.github.io.git
branch: maste

5.在_config.yml中找到Url配置

url: http://Lotharing.github.io/Lotharing.github.io
root: /Lotharing.github.io
permalink: :year/:month/:day/:title/
permalink_defaults:

6.测试是否可以部署在Github

$ hexo d

此处如果GIt报错没关系，安装个插件

$ npm install hexo-deployer-git –save

执行hexo d ，出现提交信息就成功了，可以在Github仓库页Settings 找到访问连接查看

访问连接和配置的Url有关系.url填写你创建的仓库名即xx.github.io/xx.github.io注意，我不知道为什么要这么写，但是之前测试中没有写后面重复的那一遍不显示主题

如果你也出现仓库网址为aa.github.io/xx.github.io的话url应填写aa.github.io/xx.github.io/xx.github.io
